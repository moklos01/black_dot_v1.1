process

1.research_&_the_idea

I wanted this website to feel simple at first — almost quiet. But as you interact with it, little details start to respond: a button shifts, a line moves, a color fades in. That's intentional.

I've always believed that animation, when used thoughtfully, can bring a site to life without overwhelming it. It's those tiny moments — a hover effect, a smooth transition — that make the experience feel more human.

This project was my way of exploring how subtle motion can create connection, surprise, and depth in even the most minimal design.

2.blocking_&_wireframing

Even though this was a single-page experience, structure still mattered. I started by blocking out the core sections of the landing page: hero section, programs knowledge, black_dot in numbers, skills section, process video, process detailed, my computer, ability recap, contact, footer. The focus was on hierarchy — what the user sees first, how the eye moves, and where interactions should be placed for maximum clarity and engagement.

I used quick wireframes to test layout ideas and spacing relationships. It was less about navigation patterns and more about rhythm, balance, and flow within a vertical scroll. This step helped define a clear visual cadence — establishing breathing room, grouping related content, and ensuring that key moments felt intentional even before design or motion was introduced.

3.scketch

I shifted to analog to loosen up creatively. Sketching by hand allows ideas to evolve outside of the constraints of grids and pixels. It’s where spontaneity lives — unexpected connections, layout experiments, and visual metaphors start to surface. These sketches weren’t final outputs, but they acted as a bridge between raw concepts and more polished digital design. They helped preserve a sense of play before structure took over.

4.design_&_style_frame

Using Figma, I built a focused design system to guide the look and feel of the site. I defined a clean typographic hierarchy with controlled spacing and rhythm, designed a custom icon set with a pixel-inspired aesthetic, and created rules for 3D illustrations using consistent rotation and lighting.

Instead of traditional mockups, I worked with style frames — static snapshots of key UI states — to explore contrast, mood, and transitions. These frames helped define how components would animate between states, acting as visual references for both design and motion. It was a way to align layout, interaction, and tone before writing a single line of code.

5.animation

Using Rive, I developed a set of interactive animations that added depth and responsiveness to the interface. Motion was designed to feel subtle and intentional — reinforcing interactions through smooth transitions, hover states, and micro-feedback without distracting from the content.

The style frames created earlier served as key references, helping define how elements transitioned between static and active states. These frames ensured that motion felt consistent with the visual system — same lighting, same spatial logic — just brought to life.

6.implementation

The site was built with HTML, CSS, and JavaScript, bringing together the visual system, motion design, and interaction logic into a working prototype. I focused on clean structure and direct implementation — no frameworks, just native code.

Animations were handled with a mix of Rive, CSS transitions, and JavaScript, depending on complexity and performance needs. Subtle effects used CSS, while more dynamic interactions relied on JavaScript for greater control.

The layout is currently static, with plans to expand it for responsive behavior in the next development phase.
